# 高校校园助手 APP — 开发规范与功能清单

> 基于《1、高校校园助手 APP 需求说明书》与《2、高校校园助手 APP（HarmonyOS NEXT）界面设计规范》整理。
> 当两份文档冲突时，**以设计规范为准**。本文档已排除开发排期、日期、停留时长等时间相关内容。

---

## 〇、技术背景

| 项 | 说明 |
|---|---|
| 技术栈 | HarmonyOS NEXT + ArkTS + ArkUI，Stage 模型 |
| 开发工具 | DevEco Studio（最新版），适配 Api 12 |
| 数据来源 | 全部本地数据，无后台服务器 |
| 存储 | Preferences（轻量级）+ JSON 文件 + 本地图片/音频 |
| 布局 | 线性布局（Row/Column）+ 相对布局（RelativeContainer），自适应，无硬编码尺寸 |

## 一、设计基础规范

### 1.1 配色规范

| 用途 | 色值 | 名称 | 应用场景 |
|---|---|---|---|
| 主色调 | `#1677FF` | 鸿蒙蓝 | 导航栏、Tab 栏、按钮、选中状态、标题文字 |
| 辅助色 1 | `#FF7D00` | 暖橙 | 提醒标识、待办标记、进度条、重点按钮 |
| 辅助色 2 | `#00B42A` | 清新绿 | 完成状态、成功提示、正常状态标识 |
| 背景色 | `#F5F7FA` | 浅灰 | 页面整体背景、模块分隔背景 |
| 文字主色 | `#1D2129` | 深黑 | 正文、标题、重要信息文字 |
| 文字辅色 | `#86909C` | 浅灰 | 辅助说明、占位符、时间/次要信息 |
| 边框色 | `#E5E6EB` | 淡灰 | 模块边框、输入框边框、分割线 |
| 按钮未激活态 | 背景 `#C6E2FF` / 文字 `#86909C` | — | 必填项未填完时 |
| 启动页渐变 | `#1677FF → #4096FF` | — | 启动页全屏背景 |

### 1.2 字体规范

字体：**HarmonyOS Sans**（系统默认，保证跨设备兼容）

| 文字类型 | 字号(sp) | 行高 | 字重 | 应用场景 |
|---|---|---|---|---|
| 大标题 | 32 | 1.2 倍 | 700 | 页面主标题、APP 名称 |
| 中标题 | 28 | 1.3 倍 | 600 | 模块标题、卡片标题 |
| 正文 | 26 | 1.5 倍 | 400 | 功能说明、内容文字、按钮文字 |
| 辅助文字 | 24 | 1.4 倍 | 400 | 时间、备注、占位符、底部说明 |

> 启动页 APP 名称为特例：36sp、白色、字重 700。

### 1.3 组件规范

- **圆角**：所有卡片、按钮、图片、输入框统一 `8px`（搜索栏圆角 24px 为特例）
- **间距**：元素间距 `16px`，模块间距 `24px`，页面左右边距 `20px`
- **按钮**：默认高度 `48px`，小按钮高度 `36px`，圆角 8px，文字居中
- **输入框**：高度 `48px`，圆角 8px，左右内边距 16px，默认浅灰占位符；聚焦时边框变鸿蒙蓝
- **卡片**：宽度 100%（适配页面），轻微阴影（opacity 0.05）
- **列表项缩略图**：通用 80×80，失物招领 64×64，圆角 8px
- **图标尺寸**：底部 Tab 图标 28px，九宫格图标 32px，功能列表图标 24px，音乐控制按钮 40px

### 1.4 适配规范

- 适配 HarmonyOS NEXT 主流移动端设备，屏幕尺寸覆盖 `360×780 ~ 412×915`
- 弹性布局 + 相对布局，所有元素自适应，无内容遮挡、拉伸变形
- 竖屏使用为主，避免横向滚动，支持单手操作

## 二、通用基础功能

### 2.1 应用启动与窗口管理

- **启动页**：沉浸式启动界面，展示 APP 名称、logo
- **主窗口**：设置主窗口属性，适配不同屏幕尺寸，窗口大小自适应
- **统一悬浮窗**：~~一个悬浮窗容器，可在 APP 后台时显示，内容支持在「待办信息」与「音乐播放控制」之间切换~~ ⚠️ 已废弃(2026-06-25)：当前 SDK 缺少稳定 hide/show API，音乐后台播放改由长时任务+通知栏保证
- **生命周期**：Stage 模型下应用、页面生命周期管理，保证前后台切换、页面跳转稳定

### 2.2 页面导航与路由

- **底部 Tab 导航**：首页、校园信息、学习助手、生活服务、我的 五大模块，自定义 Tab 样式
- **页面跳转**：各模块内子页面路由跳转，支持自定义转场动画和过渡效果
- **返回与刷新**：返回上一页、下拉刷新，符合移动端操作习惯；所有子页面左上角返回箭头 + 侧滑返回（HarmonyOS 原生）

### 2.3 通用组件交互

- **基础组件**：Button、Radio、Toggle、Progress、Image 等
- **事件绑定**：点击、滑动、长按等事件
- **列表组件**：展示校园资讯、课程表、失物招领等列表，支持下拉刷新、上拉加载更多

## 三、八大页面 · 功能与样式

### 3.1 启动页

**样式**
- 全屏，无状态栏、导航栏
- 背景：渐变鸿蒙蓝（`#1677FF → #4096FF`）全屏
- 中心区域：APP logo（校园建筑 + 书本，白色）→ APP 名称「高校校园助手」（36sp 白 700）→ 「HarmonyOS NEXT 版」（24sp 白 400）
- 底部：开发团队名称（22sp 白）+ 加载动画（白色环形进度条）

**功能**
- 自动跳转：首次打开→引导页；非首次→首页
- 点击屏幕任意位置跳过

**技术**：沉浸式窗口、动画组件（环形进度条、页面跳转过渡）

---

### 3.2 引导页（首次打开显示）

**样式**
- 左右滑动式，共 3 页，浅灰背景（`#F5F7FA`）
- 上半 60%：引导图（扁平风）
- 下半 40%：标题（32sp 鸿蒙蓝 700）+ 说明（26sp 主色 行高 1.5）+ 底部 3 个圆形指示器（当前页鸿蒙蓝填充，其余淡灰描边）
- 仅第 3 页有「立即进入」按钮（鸿蒙蓝，48px 高，圆角 8px，宽 80%，白字 26sp）

**三页主题**
1. 校园信息，实时掌握 — 校园公告、资讯、地图，一手了解校园动态
2. 学习助手，高效备考 — 课程表、成绩查询、图书馆检索
3. 生活服务，贴心相伴 — 失物招领、校园美食、音乐播放

**功能**
- 左右滑动切换（淡入淡出过渡）
- 点击指示器快速跳转
- 第 3 页「立即进入」→ 登录/注册页
- 第 1 页支持侧滑返回启动页

### 3.3 登录/注册页（标签切换，合并布局）

**样式**
- 浅灰背景，核心区宽 90% 居中
- 顶部标签栏：「登录」「注册」左右平分，28sp；选中鸿蒙蓝 + 底部 2px 蓝下划线，未选中文字辅色无下划线
- 表单间距 24px

**登录表单**
- 学号输入框（左学号图标，占位符「请输入你的学号」）
- 密码输入框（左密码图标，右眼睛显隐按钮，占位符「请输入你的密码」）
- 记住密码（复选框 + 文字）
- 忘记密码（鸿蒙蓝文字 → 找回密码页：学号 + 验证码）
- 登录按钮（100% 宽，48px，鸿蒙蓝，白字「登录」26sp，圆角 8px）

**注册表单**（登录表单基础上增加）
- 手机号输入框（左手机图标，占位符「请输入手机号」）
- 验证码框（右「获取验证码」暖橙小按钮，点击后 APP 在页面内直接显示验证码「您的验证码是：XXXX」，用户填入即可）
- 密码输入框、确认密码输入框
- 注册协议复选框（「我已阅读并同意《用户注册协议》」，协议文字鸿蒙蓝，点击弹窗）
- 注册按钮（同登录样式，文字「注册」）

**底部辅助区**
- 距底部 30px 居中：「暂无账号？立即注册」/「已有账号？立即登录」
- 上方间距 20px：「其他登录方式」+ 微信/QQ 圆形图标（36px，仅图标无实际授权）

**功能**
- 标签无刷新切换，表单淡入
- 输入框聚焦边框变蓝，软键盘弹出页面自适应上移不遮挡
- 按钮激活态：必填项填完→背景 `#1677FF` 白字；未填完→背景 `#C6E2FF` 文字 `#86909C`
- 前端表单验证，通过后跳首页并保存用户信息到本地 Preferences

### 3.4 首页（核心主页面）

**样式**：浅灰背景
- **沉浸式导航栏**：标题「校园助手」（30sp 鸿蒙蓝 600）+ 右侧消息铃铛（未读显示暖橙红点，→ 消息中心页）
- **顶部搜索栏**：圆角 24px 白底，左搜索图标，占位符「搜索校园信息/课程/美食」，右语音图标（仅显示）；点击跳搜索结果页
- **功能快捷入口**：3 行 3 列九宫格（共 9 个入口），圆形卡片 64px，间距 16px；图标 32px 鸿蒙蓝 + 文字 24sp 主色。入口为：校园公告、课程表、成绩查询、失物招领、校园地图、图书馆、校园美食、音乐播放、待办提醒。点击有放大微动画
- **核心模块卡片**：垂直排列，白卡宽 95%，间距 16px，轻微阴影
- **底部 Tab**：5 个平分宽，高 56px 白底无阴影，图标 28px + 文字 22sp，选中鸿蒙蓝

**核心卡片内容**
1. **校园公告**：标题「校园公告」（28sp 鸿蒙蓝 600）+ 右侧「更多」→ 公告列表页；3 条最新公告（标题 26sp / 时间 22sp / 简要 24sp 一行省略）
2. **今日课程**：无课显示「今日无课程，好好休息吧～」（26sp 辅色居中）；有课显示课名/时间/地点，当前上课用暖橙标记
3. **天气提醒**：左天气图标 48px + 温度 32sp，右天气状况 26sp + 地区 24sp；点击→天气详情页（未来 3 天，静态数据）

**功能**
- 下拉刷新（鸿蒙蓝环形进度条）
- 上拉加载更多
- 滚动时导航栏透明→白色渐变，标题色鸿蒙蓝→主色
- Tab 组件内切换，淡入淡出

### 3.5 校园信息页

**样式**
- **导航栏**：白底非沉浸，左返回箭头，中「校园信息」（30sp 主色 600），右无图标
- **分类标签栏**：横向滚动无边界，圆角按钮 36px 高、左右内边距 16px；选中蓝底白字，未选中浅灰底主色字
- **内容列表**：垂直排列，白卡宽 100%，间距 12px，内边距 16px

**分类标签**：校园公告、校园资讯、社团活动、学术讲座、校园地图（默认选中校园公告）

**通用列表项**：左缩略图 80×80（无图显默认校园图标）+ 右内容区（标题 26sp 一行省略 / 时间来源 22sp / 简要 24sp 两行省略）

**校园地图标签专属布局**
- 上方：校园平面地图（宽 100%，高 200px，圆角 8px，标注教学楼/图书馆/食堂等）
- 下方：建筑搜索框 + 核心建筑快捷入口横滚（教学楼/图书馆/食堂/宿舍）→ 建筑详情页（位置、介绍）

**功能**
- 点列表项→详情页
- 下拉刷新、上拉加载
- 地图双指缩放、单指拖动，点建筑弹出名称提示

**详情页**
- 公告详情：图文混排、收藏、分享（本地复制链接）、分类筛选（学校/学院/社团通知）
- 资讯详情：图片放大查看、文字复制

### 3.6 学习助手页

**样式**
- **导航栏**：标题「学习助手」
- **功能标签栏**：4 个标签平分宽无滚动（课程表/成绩查询/图书馆检索/学习资料），选中蓝 + 2px 下划线，未选中辅色
- **功能内容区**：标签切换对应内容，无页面跳转

**课程表**（默认选中）
- 顶部周次选择（左「上一周」/ 中「202X 年第 X 周」/ 右「下一周」，均 24sp 鸿蒙蓝）
- 主体周视图（周一至周日纵向），课程卡片用暖橙/蓝/绿区分，含课名/时间/地点
- 底部「编辑课程表」小按钮 → 编辑页
- 支持滑动切换周次，点课程卡看详情（授课老师、学分）

**成绩查询**
- 上方学号/姓名（26sp 主色）+「刷新成绩」按钮
- 下方列表（课名/成绩/绩点/考试时间），按绩点排序

**图书馆检索**
- 搜索框（占位符「请输入图书名称/作者/ISBN」）+ 搜索按钮
- 结果列表（封面/书名/作者/出版社/馆藏位置/剩余数量）+ 右侧心形收藏按钮（未收藏空心，收藏鸿蒙蓝实心）

**学习资料**
- 文件夹式卡片（图标 + 名称，如「高数课件」「计算机真题」）
- 点击进入资料列表，支持本地资料上传/查看（基础功能实现）

**课程表编辑页**：添加/修改/删除课程（课名/老师/时间/地点/学分，单选框选周次）；课程提醒 ⚠️ 已移除（HarmonyOS 限制单个应用提醒数量）；导出（本地文件保存）

**功能**：标签淡入切换、下拉刷新、关键词筛选

### 3.7 生活服务页

**音乐播放**
- 上半 60%：专辑封面（圆形 180px，播放时旋转）

- 支持悬浮窗播放（与待办共用统一悬浮窗容器）

**校园工具**（基础功能实现）
- 计算器、倒计时、快递查询

## 四、页面跳转关系

```
启动页
  ├─（首次）→ 引导页 → 登录/注册页 → 首页
  └─（非首次）→ 首页

首页（底部 Tab 切换）
  ├─ 校园信息（Tab）
  ├─ 学习助手（Tab）
  ├─ 生活服务（Tab）
  ├─ 我的（Tab）
  ├─ → 搜索结果页（搜索栏）
  └─ → 消息中心页（铃铛）

校园信息页
  ├─ → 公告/资讯详情页（列表项）
  └─ → 校园地图详情页/建筑详情页

学习助手页
  ├─ → 课程表编辑页（按钮）
  └─ → 图书详情页（列表项）

生活服务页
  ├─ → 失物招领发布页（按钮）
  └─ → 本地音乐列表页（按钮）
```

## 五、数据存储说明

全部本地存储，无后台服务器。采用**混合存储方案**，按数据类型选择最适合的存储引擎。

| 数据类型 | 存储方式 | 说明 |
|---|---|---|
| 用户信息（头像路径、昵称、学号、学院、专业、密码） | Preferences | 轻量键值，适合少量配置数据 |
| 登录态 / 首次启动标记 | Preferences | 布尔值，启动时读取 |
| 课程表 / 成绩数据 | RelationalStore | 需 SQL 查询排序、用户编辑频繁 |
| 图书馆藏 / 失物招领 / 校园美食 | RelationalStore | 需模糊搜索、分类筛选、收藏操作 |
| 天气数据 | RelationalStore | 统一数据层管理 |
| 收藏数据（公告、资讯、图书、美食） | RelationalStore | BookDao/FoodDao 持久化收藏状态 |
| 待办事项（内容、提醒时间、完成状态、分类） | Preferences | 轻量级，支持增删改查 |
| 主题模式 / 字体大小 | KVStore | 分布式键值，适合配置同步 |
| 音乐播放器（音频路径、歌曲信息） | rawfile JSON | 元数据只读，固定不变 |
| 校园公告 / 校园资讯 / 学习资料 | rawfile JSON | 非结构化文本，只读展示 |

> **数据流转**: `resources/rawfile/*.json` 存放初始模拟数据 → 首次启动通过 `DataInitializer.importToDb()` 导入 RDB（课程/成绩/图书/失物/美食/天气）→ 业务层通过 DAO 访问 RDB。公告/资讯/音乐/学习资料等只读数据不走 RDB，直接从 rawfile JSON 读取。
