## 二、优先级总览

| 级别 | 主题 | 核心产出 | 预计影响面 |
|---|---|---|---|
| **P0** | 设计规范落地 + 主流程修正 | 配色/字号常量、注册 B 方案、应用名/权限、首页与底部 5 Tab 重做 | 全局样式 + 入口页 |
| **P1** | 五大功能模块业务填充 | 学习助手 / 生活服务 / 校园信息 / 我的 完整页面与数据 | 新建大量页面与模型 |
| **P1.5** | 数据存储迁移**（新增）** | 业务数据从 JSON 迁移到 RelationalStore,DbHelper 产品级封装 | 数据层重构 |
| **P2** | 增强 | 统一悬浮窗、关系型数据库封装与本地数据、图标资源补齐 | 横切能力 + 资源 |

原则:
- **P0 必须先做**:它是「壳」,不先把样式常量与入口页对齐,P1 的新页面会继续沿用旧硬编码,越改越乱。
- **P1 内部以「学习助手」为首发**:课程表是规范明确标注的核心,且可串联代理提醒、数据库、悬浮窗三项横切能力,验证架构。
- **P1.5 数据存储迁移**:业务数据(课程表/成绩/图书/失物/美食/天气)从 JSON 文件迁移到 **RelationalStore**,利用 SQL 支持复杂查询、事务写入和索引加速,同时保留 rawfile JSON 作为初始数据导入来源。

## 五、P1.5 — 业务数据迁移：JSON → RelationalStore（✅ 2026-06-25 已完成）

> **决策背景**:课程表/成绩/图书/失物/美食/天气 等业务数据涉及用户编辑、搜索过滤、排序等操作,
> JSON 文件的「全量读→内存过滤」模式在数据量增长后效率下降,且写操作缺乏事务保障。
> 迁移到 RelationalStore 后,利用 SQL 支持复杂查询、事务写入、索引加速。
>
> **实施记录**:2026-06-25 完成。新建 AppDbHelper(6表DDL+索引)+6个DAO+DataInitializer,
> 改造 StudyTab/LifeServiceTab/WeatherDetailPage 数据源从 JsonStoreUtil 切换到 DAO。

### 5.1 迁移范围

| 数据类型 | 来源 | 目标 | 说明 |
|---|---|---|---|
| 课程表(schedule) | JSON(rawfile+沙箱) | RelationalStore | 用户可编辑,有周次/课程类型等查询条件 |
| 成绩(score) | JSON(rawfile) | RelationalStore | 需按绩点排序,后续可筛选学期 |
| 图书(books) | JSON(rawfile) | RelationalStore | 需按书名/作者/ISBN 搜索 |
| 失物招领(lostfound) | JSON(沙箱) | RelationalStore | 用户发布,需按类型/分类筛选 |
| 美食(food) | JSON(rawfile) | RelationalStore | 需按评分/食堂筛选,用户收藏 |
| 天气(weather) | JSON(rawfile) | RelationalStore | 数据量小,但迁移后统一数据层 |

**保留 JSON 不变的数据**:

| 数据类型 | 理由 |
|---|---|
| 公告(announcements) | 纯展示,非结构化文本 |
| 资讯(news) | 纯展示,非结构化文本 |
| 学习资料(study_materials) | 纯展示,目录结构 |
| 音乐(music) | 元数据,纯展示 |

### 5.2 数据库设计

**库名**:`school_app_rdb.db`

**表结构**:

```sql
-- 课程表
CREATE TABLE IF NOT EXISTS course (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  teacher TEXT,
  location TEXT,
  day_index INTEGER NOT NULL,
  start_time TEXT NOT NULL,
  end_time TEXT NOT NULL,
  start_week INTEGER NOT NULL,
  end_week INTEGER NOT NULL,
  course_type TEXT,
  color TEXT,
  credit TEXT,
  remark TEXT
);
CREATE INDEX IF NOT EXISTS idx_course_day ON course(day_index);

-- 成绩表
CREATE TABLE IF NOT EXISTS score (
  id TEXT PRIMARY KEY,
  course_name TEXT NOT NULL,
  score REAL NOT NULL,
  gpa REAL NOT NULL,
  credit TEXT,
  exam_time TEXT,
  semester TEXT
);
CREATE INDEX IF NOT EXISTS idx_score_gpa ON score(gpa DESC);

-- 图书表
CREATE TABLE IF NOT EXISTS book (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  author TEXT,
  publisher TEXT,
  isbn TEXT,
  cover TEXT,
  location TEXT,
  total_count INTEGER DEFAULT 0,
  available_count INTEGER DEFAULT 0,
  summary TEXT,
  is_favorited INTEGER DEFAULT 0
);
CREATE INDEX IF NOT EXISTS idx_book_title ON book(title);

-- 失物招领表
CREATE TABLE IF NOT EXISTS lost_found (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL CHECK(type IN ('lost','found')),
  category TEXT,
  item_name TEXT NOT NULL,
  location TEXT,
  time TEXT,
  description TEXT,
  contact_phone TEXT,
  image_path TEXT,
  publisher_id TEXT,
  publish_time TEXT,
  status TEXT DEFAULT 'pending'
);
CREATE INDEX IF NOT EXISTS idx_lf_type ON lost_found(type);
CREATE INDEX IF NOT EXISTS idx_lf_status ON lost_found(status);

-- 美食表
CREATE TABLE IF NOT EXISTS food (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  canteen TEXT,
  image_path TEXT,
  rating INTEGER DEFAULT 0,
  signature_dish TEXT,
  tags TEXT,
  is_favorited INTEGER DEFAULT 0
);

-- 天气表
CREATE TABLE IF NOT EXISTS weather (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,
  week_day TEXT,
  weather TEXT,
  temp_high INTEGER,
  temp_low INTEGER,
  icon TEXT
);
```

### 5.4 迁移流程

```
首次启动 (或 RDB 为空时):
  1. AppDbHelper 建表
  2. JsonStoreUtil.importToDb() 从 rawfile JSON 读取初始数据
  3. 逐条 INSERT 到 RDB
  4. 后续所有读写走 DAO 层

用户操作后:
  - 编辑课程表 → CourseDao.update/insert
  - 发布失物 → LostFoundDao.insert
  - 收藏美食/图书 → FoodDao/BookDao.updateFavorite

数据源切换:
  页面 aboutToAppear():
    const dao = new CourseDao();
    this.courses = await dao.queryByWeek(currentWeek);
```
