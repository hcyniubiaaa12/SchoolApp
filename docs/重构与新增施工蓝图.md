# 校园助手 APP — 重构与新增施工蓝图

> 本文档基于 `.claude/rules/校园助手APP-开发规范与功能清单.md`(以下简称「规范」)与 `.claude/rules/项目现状与规范差距分析.md`(以下简称「差距分析」),结合对现有代码的实际验证,给出可执行的**重构优先级、复用文件清单、改造文件清单、新建文件清单与执行里程碑**。
>
> 目标:把当前名为「校通Plus」的骨架项目,逐步对齐规范要求的「高校校园助手」五大模块产品形态,且全程不破坏已打通的启动 → 登录 → 首页主流程。
>
> 验证时间:2026-06-22。本文件不参与编译,仅作开发指引。

## 二、优先级总览

| 级别 | 主题 | 核心产出 | 预计影响面 |
|---|---|---|---|
| **P0** | 设计规范落地 + 主流程修正 | 配色/字号常量、注册 B 方案、应用名/权限、首页与底部 5 Tab 重做 | 全局样式 + 入口页 |
| **P1** | 五大功能模块业务填充 | 学习助手 / 生活服务 / 校园信息 / 我的 完整页面与数据 | 新建大量页面与模型 |
| **P2** | 增强 | 统一悬浮窗、JSON 存储工具与本地数据、图标资源补齐 | 横切能力 + 资源 |

原则:
- **P0 必须先做**:它是「壳」,不先把样式常量与入口页对齐,P1 的新页面会继续沿用旧硬编码,越改越乱。
- **P1 内部以「学习助手」为首发**:课程表是规范明确标注的核心,且可串联代理提醒、JSON 存储、悬浮窗三项横切能力,验证架构。
- **P2 在 P1 过程中穿插**:JSON 存储工具(`JsonStoreUtil`)其实应**提前到 P1 启动前**完成,因为五大模块都依赖它;悬浮窗和图标补齐可后置。

## 三、P0 — 设计规范落地与主流程修正

### 3.1 配色与字号常量补齐

**目标**:在 `color.json` / `float.json` 落地规范的全部色值、字号、间距、圆角常量,后续页面统一用 `$r('app.color.xxx')` / `$r('app.float.xxx')`,不再写硬编码 hex。

**新建/改造文件**:

| 文件 | 动作 | 内容 |
|---|---|---|
| `entry/src/main/resources/base/element/color.json` | 改造 | 补齐主色 `#1677FF`、暖橙 `#FF7D00`、清新绿 `#00B42A`、背景 `#F5F7FA`、文字主色 `#1D2129`、文字辅色 `#86909C`、边框 `#E5E6EB`、按钮未激活态 `#C6E2FF`、启动渐变两色等 |
